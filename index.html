第一篇博客，写一写我是如何在Github上搭建个人博客的

发表于 2022-09-09 分类于 技术文摘

1.前置准备工作

我是通过 Hexo+Github Pages 来搭建个人博客的，因此需要先安装 Node.js 和 Git 工具

官网下载：
NodeJs（建议选择长期维护版或者通过 nvm 等来安装）
Git

  • 下载完成后通过命令行来确认是否安装成功
  • 接下来安装 Hexo ，我使用的是全局安装的方式

2.创建 Github 仓库

进入 Github 的网页，登录后点击右上角+号，选择 New repository 来新建一个仓库
仓库取名建议为 【你的 Github 用户名】.github.io
仓库的访问权限设置为 Public
勾选 Add a README file，这样仓库会自动生成一个主分支，一般是 main

3.生成并设置 SSH 密钥对

如果已经配置过 Github 密钥对可以跳过此步

  • 打开 Git Bash 工具，输入
    ssh-keygen -t rsa -C "注册github账号的邮件地址"
    根据操作提示输入信息并回车来生成密钥文件

  • 生成完成后，来到.ssh 文件夹内（路径一般为 C/User/你的用户名/.ssh）
    打开 id_rsa.pub 文件，将内容全部复制

  • 点击 Github 页面右上角头像菜单里的 Setting 进入个人设置页
    点击左侧菜单 SSH and GPG keys 切换到密钥对添加页面，点击右上 New SSH key
    将刚才复制的内容全部粘贴到内容框中，标题可以自定义起

    再次打开 Git Bash 工具，输入下面的命令来炎症 SSH 密钥对是否配置成功
    ssh -T git@github.com
    如果出现
    Hi xxx. You're successfully authenticated , but Github does not ...
    则表示已经配置成功了

4.本地创建博客文件夹

新建一个文件夹，右键文件夹空白处选择 在终端中打开 或者 git bash here
建议直接使用 Vscode 打开文件夹，并选择上方菜单内的【终端】-【新建终端】，便于之后的各种编辑操作
在打开的命令行工具中，输入以下代码初始化
hexo init
初始化完成后，输入以下代码启动服务器
hexo s
页面路径默认在 http://localhost:4000/ ，浏览器访问就可以看到默认的 Hexo 页面了

6.写新的博客文章

在命令行中运行
hexo new 【文章标题】
例如
hexo new myFirstBlog
建议使用英文
在 source 文件夹下的_posts 文件夹内会生成一个以你起的文章标题命名的 md 文档
编辑该文档便可以修改博客文章，使用 md 格式来书写
写文章时可以对照hexo s运行的本地服务器页面来写，文件保存后刷新页面就可以看到最新的效果
如果需要在文章中添加图片，可以在 source 文件夹内，新建一个 images 文件夹并放在里面
使用时类似如下方式
![图片描述](/images/xxx.jpg)
或者可以在_config.yml 中，找到 post_asset_folder 并修改为 true，这样在 hexo new 的时候就会同时生成一个文章对应的专属素材文件夹

7.更换主题

可以在 Hexo 官网先找到自己喜欢的主题，然后在主题的详情页内一般有主题对应的安装方式
Hexo 官网主题

  • 这里用我现在在用的 Next 主题举例

    在博客文件夹根目录下运行
    git clone https://github.com/next-theme/theme-next-docs
    然后进入主题文件夹进行安装

    之后再回到根目录并安装

    在_config.yml 中找到 theme，改成theme: next
    之后运行hexo g hexo d主题便生效了

    找到这一段，替换上面 Schemes 部分注释的项目就是替换主题，darkmode 设置为 true 就是暗黑风格页面

如果替换主题以后hexo d，部署上去的 Github 页面有样式丢失的情况，可以在_config.yml 中找到 url 这一项，填入 github 上博客对应的地址，找到 index_generator 下的 path 项，设置为/
然后运行hexo clean清理编译缓存，再执行hexo g hexo d重新部署页面
