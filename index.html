第一篇博客，写一写我是如何在Github上搭建个人博客的

发表于 2022-09-09 分类于 技术文摘

  1. 前置准备工作
    我是通过 Hexo+Github Pages 来搭建个人博客的，因此需要先安装 Node.js 和 Git 工具

    官网下载：
    NodeJs（建议选择长期维护版或者通过 nvm 等来安装）
    Git

    • 下载完成后通过命令行来确认是否安装成功
    • 接下来安装 Hexo ，我使用的是全局安装的方式

  2. 创建 Github 仓库
    进入 Github 的网页，登录后点击右上角+号，选择 New repository 来新建一个仓库
    仓库取名建议为 【你的 Github 用户名】.github.io
    仓库的访问权限设置为 Public
    勾选 Add a README file，这样仓库会自动生成一个主分支，一般是 main

  3. 生成并设置 SSH 密钥对
    如果已经配置过 Github 密钥对可以跳过此步

    • 打开 Git Bash 工具，输入
      ssh-keygen -t rsa -C "注册github账号的邮件地址"
      根据操作提示输入信息并回车来生成密钥文件

    • 生成完成后，来到.ssh 文件夹内（路径一般为 C/User/你的用户名/.ssh）
      打开 id_rsa.pub 文件，将内容全部复制

    • 点击 Github 页面右上角头像菜单里的 Setting 进入个人设置页
      点击左侧菜单 SSH and GPG keys 切换到密钥对添加页面，点击右上 New SSH key
      将刚才复制的内容全部粘贴到内容框中，标题可以自定义起

      再次打开 Git Bash 工具，输入下面的命令来炎症 SSH 密钥对是否配置成功
      ssh -T git@github.com
      如果出现
      Hi xxx. You're successfully authenticated , but Github does not ...
      则表示已经配置成功了

  4. 本地创建博客文件夹
    新建一个文件夹，右键文件夹空白处选择 在终端中打开 或者 git bash here
    在打开的命令行工具中，输入以下代码初始化
    hexo init
    初始化完成后，输入以下代码启动服务器
    hexo s
    页面路径默认在 http://localhost:4000/ ，浏览器访问就可以看到默认的 Hexo 页面了

  5. 部署博客到 Github Pages
    首先到 Github 网页找到之前创建的仓库，点击 Code 按钮获取仓库地址
